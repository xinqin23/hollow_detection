\title{Shape vs.\ Appearance: Comparing DINOv2 and YOLOv8 on Original and Hollow Contour Images}
\author{}
\date{}

\maketitle

\begin{abstract}
We investigate how well vision models can classify images using only shape outlines (Canny edge contours) compared to full-color originals. We compare two architectures---DINOv2 (a self-supervised Vision Transformer) and YOLOv8n-cls (a lightweight CNN classifier)---on CIFAR-10 in both original and contour-only form. Both models achieve $\sim$96\% on originals but diverge sharply on contours: YOLOv8n retains 70.4\% while DINOv2 drops to 51.6\%. We further evaluate both models on a custom Overwatch character recognition dataset (14 classes), where YOLOv8n achieves 65.8\% and DINOv2 achieves 56.5\%. Across experiments, fine-tuned CNNs consistently outperform frozen self-supervised features.
\end{abstract}

\section{Introduction}

A long-standing question in visual recognition is the relative importance of shape versus texture in how neural networks classify objects~\cite{geirhos2019imagenet}. Human perception is strongly shape-biased, yet convolutional neural networks have been shown to rely more on local texture cues. Self-supervised Vision Transformers such as DINO~\cite{caron2021emerging} have been reported to develop more shape-aware representations, attending to object boundaries even without explicit supervision.

In this study, we test this claim empirically using two datasets and two contrasting architectures:

\textbf{Datasets:}
\begin{itemize}
    \item \textbf{CIFAR-10}: A standard 10-class object recognition benchmark (50k train, 10k test; native $32\times32$). We evaluate on both original images and Canny edge contour versions.
    \item \textbf{Overwatch Frames}: A custom 14-class dataset of game character screenshots (1,309 train, 336 test; native $3420\times2138$), containing both normal and partially occluded views. We train directly on this dataset to evaluate character recognition.
\end{itemize}

\textbf{Models:}
\begin{itemize}
    \item \textbf{DINOv2} (ViT-S/14): A frozen self-supervised Vision Transformer with a linear classification head trained on the target data.
    \item \textbf{YOLOv8n-cls}: A lightweight convolutional classifier, fully fine-tuned from pretrained ImageNet weights.
\end{itemize}

\section{Method}

\subsection{CIFAR-10 Dataset Preparation}

We use CIFAR-10 (50,000 training images, 10,000 test images across 10 classes). For each image, we generate a contour version by: (1) converting to grayscale, (2) applying Canny edge detection (thresholds: 100, 200), and (3) saving the binary edge map as a 3-channel PNG. All images are resized to $224 \times 224$. Figure~\ref{fig:showcase} shows example pairs.

\subsection{Overwatch Dataset Preparation}

The Overwatch dataset consists of in-game screenshots of 14 characters: Ashe, Bastion, Cassidy, Echo, Freja, Reaper, Sojourn, Soldier, Sombra, Symmetra, Torbjourn, Tracer, Venture, and Widowmaker. Each character has images from both \texttt{normal} (unobstructed) and \texttt{occluded} (partially hidden) viewpoints. We merge both conditions per character and split 80/20 into train/test sets, yielding approximately 93 training and 24 test images per class. All images are resized to $224 \times 224$. Figure~\ref{fig:ow_showcase} shows examples.

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{../results/overwatch_showcase_grid.png}
    \caption{Overwatch character samples ($224\times224$). Each cell shows one test image per character class. The 14 characters span a variety of body types, color palettes, and in-game poses.}
    \label{fig:ow_showcase}
\end{figure}

\subsection{DINOv2 Setup}

We use the \texttt{dinov2\_vits14} model from the official DINOv2 release. The ViT-S/14 backbone is frozen, and we train only a linear classifier (384-dim $\rightarrow$ $N$ classes) on top of the \texttt{[CLS]} token output. Training runs for 100 epochs with the Adam optimizer (lr = $1\times10^{-3}$) and cosine annealing. Batch size is 256 for CIFAR-10 and 64 for Overwatch. Data augmentation (random horizontal flip, color jitter) is applied for Overwatch.

\subsection{YOLOv8n-cls Setup}

We use the Ultralytics YOLOv8n-cls model, pretrained on ImageNet. All layers are fine-tuned for 30 epochs with the AdamW optimizer (auto-tuned learning rate), image size $224 \times 224$. Batch size is 256 for CIFAR-10 and 64 for Overwatch.

\subsection{Hardware}

All experiments were run on a single NVIDIA RTX 6000 Ada Generation (48 GB) with PyTorch 2.6 and CUDA 12.4.

The Overwatch dataset is substantially harder than CIFAR-10: it has more classes (14 vs.\ 10), far fewer training examples ($\sim$93 per class vs.\ 5,000), and the characters share similar in-game environments, lighting, and visual styles. YOLOv8n outperforms DINOv2 by 9.2 percentage points, consistent with the CIFAR-10 finding that full fine-tuning provides a significant advantage over a frozen backbone with a linear probe.

\section{Discussion}

\subsection{Fine-tuning vs.\ Frozen Features}

Across both datasets, YOLOv8n-cls (fully fine-tuned) consistently outperforms DINOv2 (frozen backbone + linear probe). On CIFAR-10 originals the gap is small ($<$1\%), but it widens on harder tasks: 19 points on CIFAR-10 contours and 9 points on Overwatch.

The key factor is \emph{adaptation}. DINOv2's self-supervised features are powerful general-purpose representations, but a linear probe cannot reshape them for domains that differ substantially from natural images. Full fine-tuning, as in YOLOv8n, allows the model to adapt all layers---including low-level edge detectors and mid-level texture encoders---to the specific characteristics of the target data.

\subsection{Shape Information in Contours}

The CIFAR-10 contour experiments demonstrate that Canny edge outlines retain meaningful classification signal: YOLOv8n achieves 70.4\% accuracy from shape alone. However, the $\sim$26\% drop from originals confirms that texture and color remain important complementary cues.

\subsection{Challenges of Game Character Recognition}

The Overwatch dataset presents unique challenges:
\begin{itemize}
    \item \textbf{Small training set}: $\sim$93 images per class limits what both models can learn.
    \item \textbf{Shared visual context}: All characters appear in similar game environments, so the model must rely on character-specific features rather than background cues.
    \item \textbf{Occlusion}: The inclusion of partially occluded views adds within-class variability and makes recognition harder.
\end{itemize}

Despite these challenges, YOLOv8n reaches 65.8\%, demonstrating that fine-tuned models can learn character-discriminative features even from limited data.

\section{Conclusion}

We compared DINOv2 and YOLOv8n-cls across two tasks: CIFAR-10 classification (with original and contour images) and Overwatch character recognition. YOLOv8n consistently outperforms DINOv2 across all conditions. The CIFAR-10 contour experiments show that shape alone carries substantial but incomplete classification signal, with fine-tuned models much better at exploiting it. On the Overwatch dataset, YOLOv8n achieves 65.8\% on a challenging 14-class problem with limited training data, outperforming DINOv2's 56.5\%. The primary factor across all experiments is training strategy---full fine-tuning vs.\ frozen backbone---rather than architecture (CNN vs.\ ViT).
